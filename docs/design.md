# Design Notes

## 설계 원칙

- 시간 기록을 최우선으로 한다
- 사용자는 현재 수행한 행동만 인식하면 된다
- 시작 및 종료 시각 계산은 시스템이 담당한다
- 기록 누락보다 과잉 기록을 허용한다
- 시간 공백이 발생하지 않도록 한다
- 기록 항목의 명명은 사후 처리한다

## 시간 기록 방식

- 고정 알림(Notification)을 상시 유지한다
- 잠금화면에서도 입력이 가능해야 한다

알림 입력 시:
- 입력 시각을 즉시 기록한다
- 기존 구간이 존재하면 종료 처리한다
- 새로운 구간을 즉시 시작한다

## 데이터 개념

기록은 구간이 아닌 이벤트 단위로 저장한다.

예:
- START @ 10:32
- START @ 11:05 (IDLE)
- START @ 12:10

구간 정보는 이벤트 조합을 통해 계산한다.

## UX 기준

- 기록 입력은 1초 이내에 완료되어야 한다
- 잠금 해제를 요구하지 않아야 한다
- 기록 성공 여부는 즉시 인지 가능해야 한다
