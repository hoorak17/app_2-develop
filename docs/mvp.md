# MVP Scope

## 포함 범위

- 잠금화면에서 즉시 접근 가능한 시간 기록 수단
- 버튼 입력 시점 기준의 타임스탬프 기록
- 기존 기록 구간 자동 종료 및 새 구간 시작
- 하루 단위 기록 목록 조회
- 명명되지 않은 기록 구간의 사후 정리

## 2월 9일 완성 일정 (하루 단위)

아래 일정은 2월 9일까지 하나의 완성품을 만들기 위한 과정이다.
일반적인 개발자 작업 흐름(요구사항 정리 → 설계 → 구현 → 테스트 → 안정화 → 릴리스)을
하루 단위로 쪼개 적용한다. 특히 잠금화면 접근이 막혀 있는 상황을 우선 해결한다.

### 작업 흐름(매일 공통)
- 오늘 목표/범위 확정 → 구현/수정 → 테스트/체크리스트 → 결과 기록
- 막힌 이슈는 원인 가설 → 최소 재현 → 해결/우회 방안 기록

### 상세 일정

#### 2/1 (요구사항 정리 + 잠금화면 이슈 진단)
- 포함/제외 범위 확정, MVP 성공 기준 문서 반영
- 잠금화면 접근 이슈 재현 단계 정리
- 원인 가설(권한, OS 정책, 서비스/알림 진입 등) 목록화

#### 2/2 (잠금화면 접근 해결 1차)
- 가장 가능성이 높은 접근 방식부터 시도
- 최소 구현으로 잠금화면에서 진입 가능한 경로 확보 목표
- 실패 시 원인/로그 정리 및 다음 날 시도 항목 결정

#### 2/3 (잠금화면 접근 해결 2차 + 기록 시작/전환)
- 잠금화면 진입 성공 시 기록 시작/구간 전환까지 연결
- 진입 실패 시 대체 접근(위젯/퀵타일/알림) 검토

#### 2/4 (타임스탬프 기록 + 목록 조회)
- 버튼 입력 시점 타임스탬프 저장
- 하루 단위 기록 목록 기본 화면 구성

#### 2/5 (구간 자동 종료/시작 + 데이터 안정화)
- 새로운 기록 시작 시 이전 구간 자동 종료 로직 적용
- 저장/수정 흐름 안정화 및 기본 예외 처리

#### 2/6 (명명되지 않은 구간 정리 UX)
- 미명명 구간 정리 UI/UX 완료
- 기본 빈 상태/안내 문구 추가

#### 2/7 (사용 시나리오 테스트)
- 핵심 흐름(진입 → 기록 → 전환 → 목록 → 정리) 체크리스트 점검
- 문제 발견 시 바로 수정 및 기록

#### 2/8 (안정화 + 정리)
- 오류/예외 케이스 점검 및 수정
- 문구/레이블/기본 상태 polish

#### 2/9 (최종 검수 + 배포 패키징)
- 최종 시나리오 검수 및 릴리스 노트 작성
- 사용 가이드 문서 정리

## 제외 범위

- 사전 일정 계획 기능
- 통계 및 시각화
- 자동 분석 및 추천
- PlannerMVP와의 데이터 연동
- 인공지능 기능

기록 과정이 사용자에게 부담이 되지 않는가이다.
